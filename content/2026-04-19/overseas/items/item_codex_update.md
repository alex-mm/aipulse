### 事实摘要

- OpenAI Codex 于 **04-16** 发布重大更新，能力从"代码编写"扩展至"全系统操作"
- 支持多智能体并行协作、原生网页交互、图像生成
- 新增 90+ 插件，服务全球超过 300 万开发者

### 关键细节

- **全系统操作**：视觉识别 + 点击 + 输入，可操作任意 GUI 界面，超越纯代码编写
- **多智能体并行**：多个 Codex 实例并行处理不同子任务，提升复杂项目效率
- **原生网页交互**：可直接与网页元素交互，实现浏览器自动化
- **图像生成**：集成图像生成能力，支持代码 + 视觉的混合任务
- **90+ 新插件**：大幅扩展工具生态
- **用户规模**：服务全球 300 万开发者

### 解读：这意味着什么

> 以下为编辑分析，非原文内容

- **解读**：Codex 从"代码助手"进化为"全系统操作 Agent"，与 Claude Code 的 Agentic Coding 方向形成直接竞争，但 Codex 更强调 GUI 操作能力（视觉识别 + 点击），Claude Code 更强调代码工程能力
- **解读**：300 万开发者的用户基础是 Codex 的核心护城河，90+ 新插件进一步强化生态锁定，对竞争对手形成较高的迁移壁垒
- **解读**：多智能体并行能力与 Agents SDK 的三层架构形成呼应，OpenAI 正在构建从基础设施（Agents SDK）到应用（Codex）的完整 Agentic 生态
- **解读**：海外对标：与 Anthropic Claude Code、Google Project Mariner 形成三足鼎立，AI 编程工具正在快速向"全系统 Agent"演进
